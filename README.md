# NPC Behavior Modeling using Finite Automata 🎮

This project explores the use of **Finite Automata (NFA)** to model and simulate intelligent Non-Playable Character (NPC) behaviors in video games.

## 🚀 Overview
NPCs play a critical role in enhancing gameplay by acting as enemies, allies, traders, or helpers. This project presents a structured approach to designing NPC behavior using **state-based modeling with automata theory**.

## 🎯 Objectives
- Model different NPC roles using Finite Automata
- Simulate dynamic behavior based on player interaction
- Provide a structured and scalable approach to game AI design

## 🧠 Key NPC Behaviors Modeled
- **Chaser NPC** → Tracks and pursues player
- **Trader NPC** → Handles resource exchange
- **Ally NPC** → Supports and defends player
- **Enemy NPC** → Searches, chases, and attacks

## ⚙️ Methodology
The system uses **Non-Deterministic Finite Automata (NFA)** to represent different NPC states and transitions.

Each NPC:
- Starts in an **Idle state**
- Transitions based on player actions
- Executes actions based on current state
